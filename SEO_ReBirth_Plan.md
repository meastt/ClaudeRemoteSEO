## 2. ROOT CAUSE DIAGNOSIS

Based on the data patterns, the likely causes of the decline are:

### Primary Suspects

1. **Seasonal decline (grilling/outdoor cooking):** Traffic naturally drops Nov–Feb as outdoor cooking decreases. BUT the magnitude of decline (93%) far exceeds normal seasonality (~30-50% typical for outdoor cooking niches). Something else is compounding this.

2. **Google algorithm update impact:** The sharp November drop (151 → 74 → 48 clicks in 3 weeks) suggests an algorithmic penalty or broad core update hit. Position degradation from ~14 to ~50+ confirms content was actively demoted.

3. **Content decay / staleness:** A 5.5-year-old blog likely has dozens of posts with outdated product info, broken links, old images, and stale "best of 2022/2023" references. Google increasingly penalizes outdated content.

4. **Cannibalization:** Multiple URLs competing for the same queries — see multiple boil-water pages, multiple Blackstone comparison pages, multiple vertical-vs-horizontal pages.

5. **Thin content at scale:** Many pages pulling 0 clicks despite hundreds of impressions. Positions 50-90+ indicate Google sees the content but doesn't trust it enough to rank.

### Actions Needed Before Executing This Plan

```
PREREQUISITE CHECKLIST (Agent should verify these first):
[ ] Run full site crawl (Screaming Frog or similar) to identify:
    - Broken internal/external links
    - Missing meta descriptions
    - Thin pages (< 300 words)
    - Duplicate title tags
    - Missing alt text on images
    - Orphan pages (no internal links pointing to them)
    - Redirect chains
[ ] Check Google Search Console for manual actions
[ ] Check Core Web Vitals scores (especially mobile)
[ ] Verify robots.txt is not blocking important pages
[ ] Check sitemap.xml is current and submitted
[ ] Review WordPress plugin list for bloat/conflicts
[ ] Check if any recent WordPress/theme/plugin updates broke things
[ ] Verify SSL certificate is valid
[ ] Check for any hacked/spam pages
```

17.5 Risk Mitigation
RISKS & MITIGATION:

1. RISK: Merges cause temporary ranking drops
   MITIGATION: 
     - Always 301 redirect (never 302)
     - Keep the stronger URL as canonical
     - Update all internal links to point to surviving URL
     - Request indexing immediately after redirect
     - Monitor merged pages weekly for 30 days

2. RISK: Bulk changes trigger algorithm suspicion
   MITIGATION:
     - Don't update all pages on the same day
     - Spread changes over weeks (this plan is 12 weeks for a reason)
     - Make genuine quality improvements, not just date bumps
     - Keep natural publishing cadence (2–3 updates/day max)

3. RISK: Seasonal traffic masks real progress
   MITIGATION:
     - Track position improvements separately from click/impression trends
     - Compare to same period prior year (if data available)
     - Focus on position as the leading indicator through March
     - Clicks will follow when season + positions align

4. RISK: Pruning removes pages that had hidden value
   MITIGATION:
     - Always 301 redirect pruned pages (never hard delete with 404)
     - Export full content of any page before deletion
     - Monitor pruned URLs in GSC for 30 days after redirect
     - Be conservative — when in doubt, improve rather than delete

5. RISK: New content cannibalizes existing pages
   MITIGATION:
     - Check existing content before creating any new post
     - Use different primary keywords for new vs existing content
     - Link new posts to existing related content immediately
     - Monitor GSC for any new cannibalization signals monthly

6. RISK: Competitor content outpaces refresh efforts
   MITIGATION:
     - Prioritize pages where GriddleKing has positional advantage (positions 5–15)
     - Focus on E-E-A-T signals competitors can't easily replicate
     - Build content depth through hub-spoke model
     - Maintain regular publishing cadence beyond the initial 12-week push
